System Architecture

MindBridge

A therapist-patient journaling platform that bridges the gap between sessions using AI-powered emotional analysis.

// System Flow
Client — Patient
Patient App
Journals thoughts, views AI insights, sets reminders
Flutter Android / iOS
REST
Client — Therapist
Therapist Dashboard
Reviews patient entries, mood trends, AI summaries
Flutter Web or React
Layer 2 — Identity
Firebase Authentication
Separate roles for patients and therapists. Therapist must verify patient link before accessing data.
Role-based access JWT tokens Email / Google OAuth
Layer 3a — Storage
Firestore Database
Stores journal entries, user profiles, therapist-patient links, AI analysis results
NoSQL Real-time sync Offline support
Trigger
Layer 3b — Logic
Cloud Functions
Triggered on new journal entry. Calls AI API, stores analysis, sends notifications
Node.js Auto-scaled Event-driven
Layer 4 — Intelligence
AI Analysis Engine
Detects mood, stress, emotions, patterns. Generates therapist-ready summaries and patient insights. Weekly/monthly reports auto-generated.
Claude API or Gemini API Structured prompts JSON output
Layer 5 — Engagement
Firebase Cloud Messaging
Daily reminders to patients. Alert therapist if AI detects high distress signals.
Push notifications Crisis alerts

// Key Unique Features
🔗
Therapist–Patient Link
Therapist generates an invite code. Patient enters it to consent and link accounts. Either party can unlink at any time.
🧠
Session Prep Brief
Before each session, AI auto-generates a summary of the patient's week: mood trend, key topics, flagged moments.
🚨
Crisis Detection
If AI detects distress signals (self-harm language, extreme stress), therapist gets an immediate push notification.
📊
Mood Timeline
Visual chart of emotional state across days and weeks. Both patient and therapist can view the same timeline.
🔒
Privacy Controls
Patient can mark entries as private — visible to themselves only, excluded from therapist view and AI reports.
📝
Offline Journaling
Entries saved locally when offline. Synced and analyzed when connection returns. No journal entry is ever lost.
